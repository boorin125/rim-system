# ✅ เช็คลิสต์ก่อน Commit และ Push - ภาษาไทย

**ใช้เช็คทุกครั้งก่อน commit เพื่อป้องกันความผิดพลาด!**

---

## 🔍 ก่อน Commit ทุกครั้ง

### 1. ตรวจสอบว่ามีอะไรเปลี่ยน
```bash
□ git status
  ├─ ไฟล์ถูกต้อง?
  ├─ ไม่มีไฟล์แปลกๆ?
  └─ .env ไม่อยู่ในรายการ? ✅
```

### 2. ตรวจสอบการเปลี่ยนแปลง
```bash
□ git diff
  ├─ การเปลี่ยนแปลงถูกต้อง?
  ├─ ไม่มีรหัสผ่านในโค้ด?
  ├─ ไม่มี console.log() ค้างอยู่?
  ├─ ไม่มีโค้ดที่ comment ไว้?
  └─ ไม่มี TODO ที่ไม่จำเป็น?
```

### 3. ตรวจสอบความปลอดภัย
```bash
□ ไม่มีรหัสผ่านในโค้ด
□ ไม่มี API keys
□ ไม่มีข้อมูล database
□ ไม่มี JWT secrets
□ ไฟล์ .env ไม่ถูก commit
□ ไม่มีข้อมูลส่วนตัว
```

### 4. ตรวจสอบคุณภาพโค้ด
```bash
□ โค้ด compile ได้ (npm run build)
□ ไม่มี error ของ TypeScript
□ test ยังผ่าน (ถ้ามี)
□ จัดรูปแบบถูกต้อง
□ ไม่มี import ที่ไม่ได้ใช้
```

### 5. ข้อความ Commit
```bash
□ ชัดเจนและเข้าใจง่าย
□ ใช้รูปแบบ (feat:/fix:/docs:)
□ ไม่เกิน 50 ตัวอักษร (บรรทัดแรก)
□ อธิบายว่าทำอะไรและทำไม
```

---

## 🚀 ก่อน Push ทุกครั้ง

### 1. ดึงข้อมูลล่าสุดก่อน
```bash
□ git pull
  ├─ มี conflict ไหม?
  └─ แก้ conflict แล้ว?
```

### 2. ตรวจสอบสุดท้าย
```bash
□ commit ทั้งหมดมีเหตุผล
□ ข้อความ commit ชัดเจน
□ ไม่มี commit "test" หรือ "WIP"
```

### 3. ทดสอบอีกครั้ง
```bash
□ npm run build (ยังใช้ได้?)
□ npm run start:dev (เซิร์ฟเวอร์เริ่มได้?)
□ API test ผ่าน
```

### 4. ตรวจสอบความปลอดภัยอีกรอบ
```bash
□ git log -1 (ดู commit ล่าสุด)
□ ไม่มี .env ใน commit ล่าสุด
□ ไม่เห็นรหัสผ่าน
```

---

## 📋 ขั้นตอนทำงานทั้งหมด

### ขั้นที่ 1: ตรวจสอบโค้ด
```
□ เปิด git diff
□ อ่านการเปลี่ยนทุกอัน
□ มองหา:
  ├─ ข้อมูลที่เป็นความลับ
  ├─ โค้ด debug
  ├─ การเปลี่ยนที่ยังไม่เสร็จ
  └─ ฟีเจอร์ที่ยังไม่สมบูรณ์
```

### ขั้นที่ 2: เพิ่มไฟล์
```bash
□ git add . (หรือเลือกไฟล์)
□ git status (ตรวจสอบไฟล์ที่เพิ่ม)
□ .env ไม่อยู่ในรายการ ✅
```

### ขั้นที่ 3: Commit
```bash
□ เขียนข้อความชัดเจน
□ อธิบายการเปลี่ยนแปลงถูกต้อง
□ ใช้รูปแบบที่กำหนด
```

### ขั้นที่ 4: ก่อน Push
```bash
□ git pull (ดึงข้อมูลล่าสุด)
□ แก้ conflict (ถ้ามี)
□ ทดสอบบนเครื่อง
```

### ขั้นที่ 5: Push
```bash
□ git push
□ ตรวจสอบบน GitHub
□ ดูว่าไฟล์ขึ้นถูกต้อง
```

## ⚠️ ฉุกเฉิน - Commit รหัสลับไปแล้ว?

### ถ้า commit .env หรือรหัสลับ:

**ขั้นที่ 1: ลบออกจาก Git ทันที**
```bash
git rm --cached .env
git commit -m "ลบไฟล์ .env ออกจาก git"
git push
```

**ขั้นที่ 2: เปลี่ยนรหัสผ่านทั้งหมด**
```
□ รหัสผ่าน Database
□ JWT secret
□ API keys
□ ข้อมูลลับทั้งหมด
```

**ขั้นที่ 3: แจ้งทีม (ถ้ามี)**
```
□ แจ้งทีมเรื่องข้อมูลรั่ว
□ ประสานเปลี่ยนรหัส
□ ตรวจสอบความปลอดภัย
```

## 💡 เคล็ดลับ

### 1. Commit บ่อยๆ
- Commit เล็กๆ หลายครั้ง
- ง่ายต่อการตรวจสอบ
- ง่ายต่อการย้อนกลับ

### 2. ตรวจสอบก่อน Commit
- ใช้ git diff
- เช็คทุกไฟล์
- ไม่มีเซอร์ไพรส์

### 3. ทดสอบก่อน Push
- ทดสอบอย่างรวดเร็ว
- ตรวจว่าเซิร์ฟเวอร์เริ่มได้
- เช็คฟีเจอร์สำคัญ

### 4. ข้อความชัดเจน
- ตัวเองในอนาคตจะขอบคุณ
- ทีมเข้าใจ
- ติดตามการเปลี่ยนแปลงง่าย
